路由控制表与IP包发送：

理解IP包的流动与路由表的关系很重要，无论是正常上网还是科学上网，本质上还是在地球这个局域网上遨游，各种意义上的网络体验加强，本质上就是控制路由把你的IP包转向我们想去的地方，比如转向海外的APS主机就完成了代理科学上网，代理到游戏商内网环境就实现了游戏加速，转向企业内网间接实现vpn。所以，理解路由表和IP包的关系，是理解二层网络流量分流的基础。

什么是路由表？

路由控制表中记录着网络地址与下一步应该发送至路由器的地址。在发送IP包时，首先要确定IP包首部中的目标地址，再从路由控制表中找到与改地址具有相同网络地址的记录，根据这条记录，再转发给下一个路由。如果路由表中存在多条相同网络地址的记录，就选择一个最为吻合的网络地址。所谓最吻合的网络地址，就是最匹配的那个 （最长匹配）

上面说的是普通的路由记录，一下是特殊的几个：

  • 默认路由：0.0.0.0/0或default，路由表中任何一个地址都能与之匹配，或者说，若找不到匹配的记录，直接走默认路由。

  • 主机路由：不同于源地址是网络地址和子网地址，直接一个源IP地址产生一条记录，就是主机地址。过多的主机地址当然后让路由表变得臃肿，路由器检索起来的性能就会慢下来。

  • 环回地址： 计算机内部使用一个特殊的IP地址127.0.0.1作为环回地址，程序发送给这个ip的数据包，都不会出计算机，计算机的操作系统直接扔给自己了，当然，是制定的端口。
