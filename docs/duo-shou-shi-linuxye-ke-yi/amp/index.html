让触摸板支持多手势

之前有一个古老的工具touchegg只不过多年没有更新，而且上一次使用linux的时候还是没有成功。最近发现了两种方案可以支持Ubuntu实现多手势：

  • 基于Ruby的Fusuma Github
  • 基于Gnome Extension 的插件增强 下载地址

我首先尝试的是后者，可是插件要求的GTK+版本是3.20，而我的是3.18，所以不兼容。无果，尝试前者，我就详细说一下过程：

我按照这篇教程升级了自己的GTK+到3.20，开启了前文所说的那个插件，感觉会比第一种的实现方式灵敏度更好：

值得注意的是。这个插件的运行环境要求系统前段的图像服务器为wayland session ，而不能是ubuntu17.04版本默认的xorg（Fedora25和ubuntu17.10默认wayland，并且即将到来的ubuntu18.04默认会是wayland）。所以需要手动安装一下wayland，具体做法：

sudo apt-get update
sudo apt-get install gnome-session-wayland

参考链接：
install wayland
uninstall wayland
安装完成后重启，在登录的时候切换成wayland on gnome就可以了，就可以发现刚刚的那个插件可以正常运行了。

BUG 修复

开启wayland之后虽然触摸板多手势的问题得到了解决，简直爽翻了。但是一个名明显的问题出来了——fcitx输入法不能用了，具体表现时快捷键无法呼出输入法。

解决方法如下：

编辑etc/environment文件，添加如下代码，使fcitx输入法为全局状态：

export GTK_IM_MODULE=fcitx
export QT_IM_MODULE=fcitx
export XMODIFIERS=@im=fcitx

记得修改环境变量文件后，source一下。手动点击fcitx的图标就可以看到输入法的窗口了，可以正常使用搜狗输入法啦。

让触摸板打字的时候不乱飘

在AskUbuntu搜到类似的问题，看到有人的解决方案是使用ubuntu自带的syndaemon工具，不过我试过后竟然没有生效（我都习惯了），继续找，发现原因在于自身的系统拥有触摸板的两个驱动，需要禁止其中一个就可以了。

查看当前的触摸板输入设备：

xinput list

检查是否是有两个触摸板设备，如果有两个touchpad设备，就可以采用我的方法如下：

  1. 首先，禁止其中一个驱动的方法：

编辑/usr/share/X11/xorg.conf.d/51-synaptics-quirks.conf

# Disable generic Synaptics device, as we're using
# "DLL0704:01 06CB:76AE Touchpad"
# Having multiple touchpad devices running confuses syndaemon
Section "InputClass"
        Identifier "SynPS/2 Synaptics TouchPad"
        MatchProduct "SynPS/2 Synaptics TouchPad"
        MatchIsTouchpad "on"
        MatchOS "Linux"
        MatchDevicePath "/dev/input/event*"
        Option "Ignore" "on"
EndSection

重启电脑。

  1. 然后,使用syndaemon就会生效了。
syndaemon -i 1 -d -K  //数字的单位是秒
