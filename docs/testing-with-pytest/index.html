no moudel named xxx (是我盛放flask程序的主目录)

本以为，在test目录下，新建一个空的__init__.py，然后终端键入pytest -v，嘿，运行正常，好使，本以为这就完事了，紧接着看到下面有位小哥评论：

然后，我的心流：15分钟内找不到最优雅的方案，先用__init__.py满足需求，毕竟距离毕业不多了，不多的青春可以放肆。干！

然后，找到了这篇文章: Installation and Getting Started

然后找到了第一个方法：

python3 -m pytest [-v]

结果就好使了，没有报这样之前的找不到模块的错误。但是我今天上午用的其他人的代码可以开森的直接使用pytest -v,而我的为什么不可以呢？

总结

刚开始，也走了一些弯路，逃出了测试的范围：比如，我想实现在单元测试中实现对json的校验，看过这篇文章后,作者的思路是定义一个json Schema来实现和reponse中的json data对比，但是我认为这样有一个不好的地方——数据源不唯一，因为model本身也是一份对于数据结构声明的数据，试想，在以后的字段改变中，基于接口的新需求，需要修改model的定义，而使用了flask_marshmallow会自动帮我把修改映射到对应的ModelSchema上，那面向单元测试的json schma呢？还要麻烦的同步修改，所以我没有采用那篇博客说明的方法。

然后，我想到了，其实对于josn数据结构的校验不正应该是业务代码负责的事情么，如果数据结构不对，后台返回报错message不就行了么，单元测试中用非法的数据并断言服务器会返回报错message不就行了么。沉迷测试不知归路，倒逼自己写好业务的数据检查，嗯嗯，若有所思。
