具体怎么践行API First？

by luke

2018-03-25

现在都在谈API First和Test First，我看，都没有Coffee First来的实在，哈哈。

践行API First理念，是在开发之前就要完成系统后台的接口设计，包括：

  • 接口资源划分各个组URL长什么样子
  • 接口包含哪些http方法
  • 接口的数据格式
  • 接口的权限验证以及角色的划分
  • 接口的频率设定

有了上面说的这些数据，几乎前端和后台就可以分开干活了，只要彼此严格践行同一套API设计范式就可以了。

话虽然说得容易，但要是是结构优良的整合上述说的设计草案，具体的形式是什么呢？可别天真的以为一个word文档或者ppt就可以了，这种方式还是不足够高效率，不足够统一，毕竟没有严格的模式限定，人与人之间对于文字的理解是不一样的。

“当我们在谈接口设计的时候，我们在谈些什么”

所以统一范式的需求与文档化的需求结合在一起，一个很优良的解决方案产生了，就是——API Blueprint，谷歌之后官网文档半个小时可以掌握，是的，你明白的，用咱们都喜爱的Markdown文档编写API文档，可能我这样说还是不太准确，所以务必看官网。

使用API Blueprint方法编写API文档，对应的工具首选是Apiary，一个web网站，可以实时编辑，交互体验这个新的方法去设计API。第一次使用实在本机VS code写的，直接粘上去，结果出来28个warning，吓死我了。

尝试下来，周边的还会了解到MSON——用Markdown语义化的描述对象。MSON的官方文档

一些容易犯错的地方

  1. post请求字段的描述：github上的文档

  2. response描述： 注意分层级，内层要比外层多一个tab，而且换行无间隔。

  3. Data Structures的定义，放在组定义的最后，标题与接口定义同级。

  4. Attributes与Parameters容易混用: 记住一点，URI里的变量定义与Parameters严格对等，Attributes就是发送请求带的数据，一般是object，当然也可以指定为arrary[YOUR_OBJECT]，里面是你定义的对象。

说了这么多，举一个栗子吧.

API blueprint描述一个需要jwt验证的接口：

## OAuth token exchange [/oauth/token]


### Exchange credentials for token [POST]

获取最新的token

+ Request
    + Attributes (OAuth grant request)
    + Headers

            Authorization: Basic ABCDEF

+ Response 200 (application/json)
    + Attributes (OAuth valid response)

+ Request
    + Attributes (OAuth jwt-bearer grant request)

+ Response 200 (application/json)
    + Attributes (OAuth valid response)

##  OAuth 2 Bearer Token protected resource [/protected]

举一个需要验证的接口的栗子。

### Status [GET]
+ Request
    + Headers

            Authentication: Bearer JWT

+ Response 200 (application/json)
    + Attributes (Server response)

## Data Structures

### `OAuth grant request` (object)
+ `grant_type`: `client_credentials` (string, required)

### `OAuth jwt-bearer grant request` (object)

+ `grant_type`: `urn:ietf:params:oauth:grant-type:jwt-bearer` (string, required)
+ assertion: `eyJhbGciOiJIUzI1NiJ9.e30.XmNK3GpH3Ys_7wsYBfq4C3M6goz71I7dTgUkuIa5lyQ` (string, required)

### `OAuth valid response` (object)
+ `access_token`: `eyJhbGciOiJIUzI1NiJ9.e30.XmNK3GpH3Ys_7wsYBfq4C3M6goz71I7dTgUkuIa5lyQ` (string, required) - valid JWT
+ scope: all (string, required) - scopes of current token
+ `expires_in`: 300 (number, required)
+ `token_type`: Bearer (string, required)

### Server response (object)
+ status: ok (string, required)
