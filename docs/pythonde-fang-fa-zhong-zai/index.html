Python的方法重载

python则比较特殊，它本身是动态语言，方法的参数是没有类型的，当调用传值的时候才确定参数的类型。

所以python的重载实现方式使用了Python的可变长度的参数这一特性：

def func_name(*tuple_grp_nonkw_args, **dict_grp_kw_args)

其中tuple_grp_nonkw_args是一个元组容器用来体现非关键字参数组，而dict_grp_kw_args是一个字典容器（本质是一个字典）用来体现装有关键字参数。

为了区分非关键字参数组和有关键字参数组，分别使用*和**体现。

再结合一般正规参数与默认参数，牛逼的python函数声明可以通配表现出这个样子：

def function_name([formal_args,][default_arg=default_value,][*vargst,][,**vargsd])

我感觉这样的优点是：简单明了，一旦了解了这种方式，就容易理解，而且代码复用比较好。区别Java或C#那样，写好几个同名函数，重复很多代码。记得自己以前学Java的时候，很多次出现本身没有考虑的重载情况，还要回去写特定的重载方法。当然啦，指定准确的对象类型，效率肯定要快一些。
